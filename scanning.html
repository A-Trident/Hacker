Scanning Networks

Network scanning is the second phase of ethical hacking. It involves identifying live hosts, open ports, running services, operating systems, and vulnerabilities on the target network.

TCP Scanning UDP Scanning Stealth Scanning

What is Network Scanning?

Understanding the objectives and types of network scanning.

Objectives

Discover live hosts and IP addresses, identify open & closed ports, detect running services and their versions, determine operating systems (OS fingerprinting), and find potential vulnerabilities.

Types of Scanning

• Port Scanning — Identify open/closed ports
• Network Scanning — Discover live hosts
• Vulnerability Scanning — Find weaknesses
• Service Scanning — Detect running services
• OS Fingerprinting — Identify operating systems

TCP/IP Basics

Understanding the TCP three-way handshake is crucial:
1. SYN → Client sends SYN packet
2. SYN/ACK ← Server responds
3. ACK → Client confirms connection
This handshake forms the basis of most scanning techniques.

Port States

Understanding what different port states mean during scanning.

State Description Implication
Open A service is actively listening on the port Potential entry point — service can be probed further
Closed Port is accessible but no service is listening Host is alive but the port is not in use
Filtered A firewall or filter is blocking the port Cannot determine if port is open or closed
Unfiltered Port is accessible but state cannot be determined ACK scan result — needs further investigation
Open|Filtered Cannot distinguish between open and filtered No response received — common with UDP scans
Closed|Filtered Cannot distinguish between closed and filtered Rare — seen in IP ID idle scans

Scanning Methodology

Step-by-step approach to network scanning.

1
Host Discovery (Ping Sweep)

Identify live hosts on the network using ICMP ping, ARP requests, TCP SYN/ACK probes, or UDP probes. Tools: Nmap, Angry IP Scanner, fping.

2
Port Scanning

Scan discovered hosts for open, closed, and filtered ports. Use TCP Connect, SYN Stealth, FIN, XMAS, NULL, and UDP scans depending on stealth requirements.

3
Service & Version Detection

Determine what services are running on open ports and their versions. This helps identify known vulnerabilities associated with specific versions.

4
OS Fingerprinting

Identify the operating system running on the target using TCP/IP stack fingerprinting. Active OS detection sends crafted packets; passive listens to traffic patterns.

5
Vulnerability Scanning

Scan for known vulnerabilities using automated tools like Nessus, OpenVAS, or Qualys. Compare discovered services and versions against vulnerability databases.

Scan Types

Different scanning techniques and when to use them.

TCP Connect Scan

Completes the full TCP three-way handshake. Most reliable but easily detected by firewalls and IDS.

-sTTCP
nmap -sT target.com
SYN Stealth Scan

Sends SYN packets without completing the handshake. Faster and stealthier than TCP Connect. Default Nmap scan.

-sSStealth
nmap -sS target.com
UDP Scan

Scans for open UDP ports. Slower than TCP scans because UDP is connectionless. Essential for finding DNS, SNMP, DHCP.

-sUUDP
nmap -sU target.com
FIN Scan

Sends a FIN packet. Open ports ignore FIN; closed ports respond with RST. Useful for bypassing stateless firewalls.

-sFStealth
nmap -sF target.com
XMAS Scan

Sets FIN, PSH, and URG flags simultaneously (lit up like a Christmas tree). Stealthy against some firewalls.

-sXStealth
nmap -sX target.com
NULL Scan

Sends a packet with no flags set. Open ports don't respond; closed ports send RST. Bypasses some firewalls.

-sNStealth
nmap -sN target.com
Idle/Zombie Scan

Uses a third-party "zombie" host to scan the target, making the scan completely untraceable to the attacker.

-sIStealth
nmap -sI zombie_host target.com
Aggressive Scan

Combines OS detection, version detection, script scanning, and traceroute in one scan. Noisy but comprehensive.

-AAggressive
nmap -A target.com
ACK Scan

Determines if ports are filtered or unfiltered. Doesn't identify open ports but maps firewall rulesets.

-sATCP
nmap -sA target.com

Nmap Cheat Sheet

Essential Nmap commands for everyday use.

Always obtain written authorization before scanning any network or system. Unauthorized scanning is illegal and may result in criminal charges.

Other Scanning Tools

Additional tools used in the scanning phase.

Nessus

Industry-leading vulnerability scanner. Identifies misconfigurations, missing patches, default credentials, and known CVEs.

Commercial
OpenVAS

Open-source vulnerability scanner (Greenbone). Full-featured alternative to Nessus with extensive vulnerability database.

Open Source
openvas-start
Masscan

Fastest port scanner — scans the entire Internet in under 6 minutes. Ideal for large-scale host discovery.

Fast
masscan -p1-65535 192.168.1.0/24 --rate=1000
Nikto

Web server vulnerability scanner. Checks for dangerous files, outdated server software, and misconfigurations.

Web Scanner
nikto -h http://target.com
Hping3

TCP/IP packet assembler and analyzer. Craft custom packets for firewall testing, port scanning, and OS fingerprinting.

Advanced
hping3 -S target.com -p 80 -c 3
Angry IP Scanner

Fast and simple network scanner with GUI. Pings each IP address, resolves hostnames, scans ports, and detects MAC addresses.

GUI Tool

Countermeasures

How to defend against network scanning attacks.

Network Defenses

• Configure firewalls to block unauthorized scans
• Implement Intrusion Detection Systems (IDS/IPS)
• Close unnecessary ports and disable unused services
• Use port knocking for sensitive services
• Implement network segmentation and VLANs
• Deploy honeypots to detect scanning activity
• Rate-limit incoming connections

System Hardening

• Keep systems patched and updated regularly
• Change default banners and service fingerprints
• Disable ICMP echo responses where appropriate
• Use TCP wrappers to restrict service access
• Implement strong access control lists (ACLs)
• Regularly audit open ports and running services
• Use encrypted protocols (SSH, HTTPS, SFTP)
