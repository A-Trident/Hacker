Man In The Middle

A Man-in-the-Middle (MITM) attack occurs when an attacker secretly intercepts and potentially alters communication between two parties who believe they are communicating directly with each other.

Interception SSL/TLS Attacks ARP Spoofing
Legal Warning

Intercepting network communications without authorization is a serious criminal offense under wiretapping and computer fraud laws worldwide. MITM techniques must ONLY be used in authorized penetration testing engagements with proper written consent. This content is strictly for educational and defensive purposes.

How MITM Works

Understanding the interception and relay of communications.

Victim (Alice)

Thinks she's talking to Bob

Intercepted Modified
Attacker (Eve)

Intercepts & relays

Forwarded Captured
Server (Bob)

Thinks he's talking to Alice

Normal Communication

Alice → Bob

• Direct encrypted connection (TLS/SSL)
• Certificate verified by trusted CA
• Data integrity maintained end-to-end
• No third party can read or modify data
• Authentication verified on both ends

MITM Communication

Alice → Eve → Bob

• Attacker sits between both parties
• Intercepts, reads, and modifies messages
• Presents fake certificates to both sides
• Both parties unaware of interception
• Credentials and data stolen in real-time

MITM Attack Types

Different techniques used to perform Man-in-the-Middle attacks.

ARP Spoofing/Poisoning

Sends falsified ARP messages on a LAN to link attacker's MAC address with the IP of a legitimate host, redirecting traffic through the attacker.

LAN Attack
arpspoof -i eth0 -t victim_ip gateway_ip
DNS Spoofing/Poisoning

Corrupts the DNS cache to redirect domain name resolution to malicious IP addresses, sending users to fake websites.

Redirection
dnsspoof -i eth0 -f hosts.txt
SSL Stripping

Downgrades HTTPS connections to HTTP by intercepting the initial redirect, allowing the attacker to see all traffic in plaintext.

Crypto Attack
sslstrip -l 8080
DHCP Spoofing

Sets up a rogue DHCP server to assign clients the attacker's IP as the default gateway, routing all traffic through the attacker.

LAN Attack
Evil Twin / Rogue AP

Creates a fake WiFi access point mimicking a legitimate one. Victims connect and all their traffic passes through the attacker.

WiFi Attack
Email Hijacking

Compromises email accounts or mail servers to intercept, read, modify, and forward email communications between parties.

Passive
SSL/TLS Interception

Uses forged or stolen certificates to intercept encrypted HTTPS traffic. Tools like mitmproxy present fake certs to the victim.

Crypto Attack
ICMP Redirect

Sends forged ICMP redirect messages to alter the victim's routing table, forcing traffic through the attacker's machine.

Network
Session Hijacking

Steals or predicts session tokens/cookies to take over an authenticated user's session without needing their credentials.

Session Attack

MITM Tools

Tools used for MITM testing in authorized engagements.

Ettercap
Comprehensive MITM attack suite

Ettercap is a comprehensive suite for MITM attacks. It supports ARP poisoning, ICMP redirect, DHCP spoofing, port stealing, and more.

Key Features:
  • Active and passive dissection of protocols (HTTP, FTP, SSH, HTTPS)
  • Content filtering and injection on the fly
  • Plugin architecture for extending functionality
  • GUI and command-line interfaces
Commands:
ettercap -G
ettercap -T -q -i eth0 -M arp:remote /victim_ip// /gateway_ip//
ettercap -T -q -i eth0 -P dns_spoof -M arp /victim_ip// /gateway_ip//
Bettercap
Modern MITM framework (Ettercap successor)

Bettercap is the Swiss Army knife for network attacks and monitoring. It's the modern successor to Ettercap with a powerful interactive web UI.

Capabilities:
  • ARP spoofing, DNS spoofing, DHCP spoofing
  • WiFi reconnaissance and deauthentication
  • BLE (Bluetooth Low Energy) device scanning
  • HTTP/HTTPS proxy with SSL stripping
  • Packet sniffing and credential harvesting
  • Caplet scripting for automation
Commands:
bettercap -iface eth0
net.probe on
set arp.spoof.targets victim_ip
arp.spoof on
net.sniff on
mitmproxy
Interactive HTTPS proxy

mitmproxy is an interactive TLS-capable intercepting HTTP proxy. Perfect for analyzing and modifying HTTPS traffic in real-time.

Features:
  • Interactive console with request/response inspection
  • Web-based interface (mitmweb)
  • Python scripting API for automation
  • SSL/TLS interception with auto-generated certificates
  • Replay, modify, and inject HTTP traffic
Commands:
mitmproxy --mode transparent --showhost
mitmweb -p 8080
mitmdump -w traffic.log
Wireshark & Packet Analysis
Network protocol analyzer for captured traffic

After intercepting traffic via MITM, Wireshark is used to analyze captured packets for credentials, session tokens, and sensitive data.

Useful Wireshark Filters:
http.request.method == "POST"
tcp.port == 80 || tcp.port == 443
ftp.request.command == "PASS"
http.cookie contains "session"
dns.qry.name contains "target.com"

Real-World Scenarios

Common situations where MITM attacks occur.

Public WiFi Coffee Shop Attack

An attacker sets up an "Evil Twin" access point named "FreeWiFi_CoffeeShop" in a café. Unsuspecting users connect and all their HTTP traffic, login credentials, and session cookies are intercepted by the attacker. HTTPS connections may be downgraded via SSL stripping.

Corporate Network ARP Poisoning

An insider or compromised device on a corporate LAN performs ARP spoofing to position itself between employees and the default gateway. All internal HTTP traffic, email credentials, and intranet sessions are captured. Even VoIP calls can be intercepted.

Banking SSL Interception

An attacker compromises a user's machine and installs a rogue CA certificate. All HTTPS traffic, including online banking sessions, is decrypted, modified (e.g., changing payment recipient details), and re-encrypted before forwarding to the legitimate bank server.

Mobile App API Interception

A tester configures mitmproxy as the device's proxy and installs the mitmproxy CA certificate. All API calls from mobile apps are intercepted, revealing API keys, authentication tokens, user data, and insecure endpoints.

Attack Methodology

Step-by-step process of a MITM attack (for authorized testing).

1
Position on the Network

Connect to the same network as the target. For WiFi attacks, create a rogue AP. For LAN attacks, connect to the same subnet or VLAN.

2
Enable IP Forwarding

Enable packet forwarding so intercepted traffic is relayed to the intended destination, maintaining connectivity for the victim.

3
Perform ARP Spoofing

Send gratuitous ARP replies to poison the victim's and gateway's ARP cache, redirecting traffic through the attacker's machine.

4
Intercept & Capture Traffic

Use sniffing tools (Wireshark, tcpdump) to capture traffic. For HTTPS, deploy SSL stripping or proxy with forged certificates.

5
Extract & Analyze Data

Parse captured traffic for credentials, session tokens, cookies, API keys, and sensitive information. Analyze protocols for weaknesses.

Quick MITM Setup (Linux)
Essential commands for authorized MITM testing
Step 1: Enable IP Forwarding
echo 1 > /proc/sys/net/ipv4/ip_forward
Step 2: ARP Spoofing (Terminal 1)
arpspoof -i eth0 -t victim_ip gateway_ip
Step 2b: ARP Spoofing (Terminal 2)
arpspoof -i eth0 -t gateway_ip victim_ip
Step 3: Capture Traffic
tcpdump -i eth0 -w capture.pcap host victim_ip
Step 4: SSL Strip (Optional)
iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 8080
sslstrip -l 8080
Step 5: Credential Sniffing
dsniff -i eth0

Detecting MITM Attacks

Signs that indicate a Man-in-the-Middle attack may be in progress.

IndicatorDescriptionDetection Method
Certificate Warnings Browser shows SSL certificate errors or warnings about untrusted certificates User awareness, HSTS enforcement
ARP Table Changes Multiple IP addresses mapping to the same MAC address arp -a, ARP monitoring tools
Increased Latency Noticeable delay in network communication due to traffic relay Network monitoring, ping analysis
HTTP Downgrade Websites loading over HTTP instead of HTTPS Browser indicators, HSTS preloading
Duplicate MAC Same MAC address appearing for different IPs on the network Network switch port security, IDS
DNS Anomalies DNS responses pointing to unexpected IP addresses DNS monitoring, DNSSEC validation
Unexpected Disconnections Frequent WiFi disconnections and reconnections WiFi monitoring, deauth detection

Countermeasures

How to protect against Man-in-the-Middle attacks.

Encryption

• Use HTTPS everywhere (HSTS preloading)
• Implement TLS 1.3 with strong cipher suites
• Enable certificate pinning in mobile apps
• Use VPN on untrusted networks
• Deploy end-to-end encryption for messaging
• Enable DNSSEC for DNS integrity

Network Security

• Enable Dynamic ARP Inspection (DAI)
• Configure DHCP snooping on switches
• Use 802.1X port authentication
• Implement network segmentation
• Deploy IDS/IPS for anomaly detection
• Monitor for rogue access points

User Awareness

• Verify SSL certificate validity in browser
• Avoid using public WiFi for sensitive tasks
• Watch for certificate warnings — don't ignore them
• Use password managers (detect phishing domains)
• Enable Multi-Factor Authentication
• Train employees on MITM awareness
