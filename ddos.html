DDoS Attacks

A Distributed Denial of Service (DDoS) attack overwhelms a target server, service, or network with a flood of Internet traffic from multiple sources, rendering it unavailable to legitimate users.

Volumetric Protocol Application Layer
Serious Legal Warning

Launching DDoS attacks is a serious criminal offense in virtually every jurisdiction worldwide. Penalties include heavy fines and imprisonment. This content is provided purely for educational and defensive purposes. Understanding DDoS attacks is essential for building effective defenses. Never attack systems without explicit authorization.

DoS vs DDoS

Understanding the difference between Denial of Service and Distributed Denial of Service.

DoS (Denial of Service)

• Single source — Attack originates from one machine
• Lower bandwidth — Limited by single connection speed
• Easier to block — Block the source IP address
• Simpler execution — Can be launched from a single script
• Less effective — Against modern infrastructure
• Traceable — Easier to identify the attacker

DDoS (Distributed Denial of Service)

• Multiple sources — Attack from thousands of compromised machines (botnet)
• Massive bandwidth — Combined bandwidth of all bots
• Hard to block — Cannot simply block one IP
• Complex execution — Requires botnet infrastructure
• Highly effective — Can take down major services
• Difficult to trace — Traffic comes from legitimate-looking sources

Botnets Explained
The backbone of DDoS attacks

A botnet is a network of compromised computers (bots/zombies) controlled by an attacker through a Command and Control (C&C) server.

How Botnets Are Built:
  • Malware distribution — Through phishing emails, drive-by downloads, or infected software
  • IoT exploitation — Scanning for IoT devices with default credentials (Mirai botnet)
  • Vulnerability exploitation — Automated scanning and exploitation of unpatched systems
  • Worm propagation — Self-replicating malware spreading across networks
Notable Botnets:
  • Mirai — IoT botnet that attacked Dyn DNS (2016), took down Twitter, Netflix, Reddit
  • Emotet — Banking trojan turned botnet-as-a-service infrastructure
  • Mēris — Attacked Yandex with 21.8 million requests per second (2021)
  • Mantis — Record 26 million HTTPS requests per second (2022)

Attack Categories

The three main categories of DDoS attacks based on OSI model targeting.

Volumetric Attacks (Layer 3/4)
Flood the target's bandwidth

Volumetric attacks aim to consume all available bandwidth between the target and the Internet by sending massive amounts of traffic.

Common Volumetric Attacks:
  • UDP Flood — Sends massive UDP packets to random ports. Target must process and respond with ICMP "Destination Unreachable" packets, consuming resources.
  • ICMP Flood (Ping Flood) — Overwhelms target with ICMP Echo Request packets requiring responses for each packet.
  • DNS Amplification — Sends small DNS queries with spoofed source IP. DNS servers respond with much larger replies to the target (amplification factor ~28-54x).
  • NTP Amplification — Exploits NTP monlist command for amplification factor of ~556x.
  • Memcached Amplification — Exploits open Memcached servers for amplification factor up to ~51,000x (largest known amplification).
  • SSDP Amplification — Exploits UPnP-enabled devices for ~30x amplification.
Measured in: Gbps or Tbps
Protocol Attacks (Layer 3/4)
Exploit protocol weaknesses

Protocol attacks exploit weaknesses in network protocol implementations to consume server resources or intermediate equipment like firewalls and load balancers.

Common Protocol Attacks:
  • SYN Flood — Sends a flood of SYN packets without completing the TCP handshake, filling up the target's connection table (half-open connections).
  • ACK Flood — Sends massive ACK packets that the server must process and look up in its connection table.
  • SYN-ACK Flood — Sends spoofed SYN-ACK packets causing the target to send RST for each, consuming CPU.
  • Ping of Death — Sends oversized or malformed ICMP packets that crash the target system.
  • Smurf Attack — Sends ICMP requests with the target's spoofed IP to a broadcast address, causing all hosts to reply to the target.
  • Fragmentation Attack — Sends fragmented packets that the target cannot reassemble, consuming memory.
Measured in: Packets per second (pps)
Application Layer Attacks (Layer 7)
Target web applications & services

Application layer attacks target the actual web server or application, making them harder to detect because they mimic legitimate user traffic.

Common Application Layer Attacks:
  • HTTP Flood — Sends a massive number of HTTP GET or POST requests that appear legitimate. Very difficult to distinguish from real traffic.
  • Slowloris — Opens many connections and sends partial HTTP headers very slowly, keeping connections open and exhausting the server's connection pool.
  • RUDY (R-U-Dead-Yet) — Sends POST requests with extremely long content-length headers, sending data one byte at a time.
  • DNS Query Flood — Sends a flood of DNS queries for non-existent domains, overwhelming the DNS server.
  • WordPress Pingback — Exploits WordPress XML-RPC pingback feature to generate massive traffic.
  • HTTPS Flood — Encrypted HTTP flood that is harder to inspect and filter due to TLS encryption overhead.
Measured in: Requests per second (rps)

Testing & Analysis Tools

Tools used for DDoS stress testing and traffic analysis (authorized testing only).

hping3

TCP/IP packet assembler. Used for SYN flood testing, firewall testing, and advanced port scanning on authorized networks.

Protocol Testing
hping3 -S --flood -V -p 80 target_ip
LOIC (Low Orbit Ion Cannon)

Open-source network stress testing tool. Sends TCP, UDP, or HTTP requests to stress test a target (authorized use only).

Stress Test
Slowloris

Layer 7 denial of service tool that uses minimal bandwidth to keep connections open by sending partial HTTP headers.

App Layer
slowloris target_ip -p 80 -s 500
Wireshark

Network protocol analyzer. Essential for detecting DDoS traffic patterns, analyzing packet captures, and identifying attack signatures.

Analysis
wireshark -i eth0 -f "tcp port 80"
tcpdump

Command-line packet analyzer. Capture and analyze network traffic to detect DDoS patterns and anomalous traffic.

Analysis
tcpdump -i eth0 -nn 'tcp[tcpflags] == tcp-syn'
NetFlow/sFlow Analyzers

Monitor network flow data to detect volumetric anomalies, traffic spikes, and unusual patterns indicative of DDoS attacks.

Monitoring

DDoS Detection

How to identify if you're under a DDoS attack.

IndicatorDescriptionDetection Method
Traffic Spike Sudden unexplained surge in network traffic NetFlow analysis, bandwidth monitoring
High CPU/Memory Server resources maxed out without legitimate cause System monitoring tools (Nagios, Zabbix)
Connection Timeouts Legitimate users unable to connect or experiencing timeouts Application logs, user reports
Single IP Flooding Unusually high number of requests from single source Access logs, firewall logs
Geographic Anomaly Traffic from unexpected geographic regions GeoIP analysis, CDN dashboards
Protocol Anomaly Unusual protocol distribution (e.g., all UDP traffic) Packet capture analysis (Wireshark, tcpdump)
HTTP Error Surge Massive increase in 5xx server errors Web server logs, monitoring dashboards

Defense & Countermeasures

Multi-layered strategies for DDoS mitigation.

Cloud-Based Protection

• Cloudflare — Global CDN with DDoS protection
• AWS Shield — Managed DDoS protection for AWS
• Akamai Prolexic — Enterprise DDoS mitigation
• Azure DDoS Protection — Microsoft cloud defense
• Google Cloud Armor — WAF + DDoS protection
• Anycast routing to distribute attack traffic globally

Network-Level Defense

• Rate limiting — Limit requests per IP per second
• Blackhole routing — Route attack traffic to null
• ACL filtering — Block known malicious IPs
• BGP Flowspec — Programmatic traffic filtering
• Scrubbing centers — Clean traffic before delivery
• Over-provisioning bandwidth capacity

Application-Level Defense

• WAF — Web Application Firewall rules
• CAPTCHA challenges — Distinguish bots from humans
• JavaScript challenges — Browser verification
• Connection timeouts — Aggressive timeout settings
• Load balancing — Distribute traffic across servers
• IP reputation scoring and geo-blocking

Linux iptables DDoS Mitigation Rules
Basic firewall rules for DDoS defense
Drop Invalid Packets
iptables -t mangle -A PREROUTING -m conntrack --ctstate INVALID -j DROP
Block SYN Flood
iptables -A INPUT -p tcp --syn -m limit --limit 1/s --limit-burst 3 -j ACCEPT
Limit ICMP (Ping Flood Protection)
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT
Rate Limit Connections per IP
iptables -A INPUT -p tcp --dport 80 -m connlimit --connlimit-above 50 -j REJECT
Drop Fragmented Packets
iptables -A INPUT -f -j DROP
Block XMAS Packets
iptables -A INPUT -p tcp --tcp-flags ALL ALL -j DROP
Block NULL Packets
iptables -A INPUT -p tcp --tcp-flags ALL NONE -j DROP

Rate limiting & connection control

Incident Response Plan

What to do when you're under a DDoS attack.

Immediate Actions

1. Confirm it's a DDoS attack (not just a traffic spike)
2. Activate DDoS mitigation service (Cloudflare, AWS Shield)
3. Notify your ISP/hosting provider
4. Enable rate limiting and geo-blocking
5. Scale up infrastructure if possible
6. Document the attack (logs, traffic captures)

Post-Attack Actions

1. Analyze attack vectors and traffic patterns
2. Update firewall rules based on findings
3. Review and improve DDoS response plan
4. Consider upgrading DDoS protection services
5. Report the attack to law enforcement (IC3, CERT)
6. Conduct post-mortem with the team
